# Контекст проекта: Bnovo API

## Что это

Скрипт для работы с API эко-отеля «Золотой Берег» через систему управления Bnovo PMS.
Используется для получения данных о бронированиях, анализа загрузки и расчёта метрик.

## Учётные данные

- **ID пользователя:** 107993
- **API ключ:** хранится в переменной окружения `BNOVO_PASSWORD` или передаётся через `--password`
- **Кэш токена:** `.bnovo_token.json` (создаётся автоматически, срок жизни берётся из `exp` JWT)
- **Важно:** API-ключ / пароль не сохраняется в `.bnovo_token.json`; там допустимы только `access_token` и `user_id`

## Как запускать скрипт

```bash
# Последние 14 дней (если токен валиден — берётся из кэша)
python3 bnovo.py

# Указать период
python3 bnovo.py --from 2026-03-01 --to 2026-03-31

# Первый запуск / принудительная авторизация
python3 bnovo.py --id 107993 --password "ключ" --from 2026-03-01

# Через переменные окружения
export BNOVO_ID=107993
export BNOVO_PASSWORD="ключ"
python3 bnovo.py

# Фильтр по типу выборки (по фактическому заезду/выезду, как шахматка)
python3 bnovo.py --from 2026-03-30 --to 2026-04-05 --type checkmate

# Сырой JSON (для дальнейшей обработки)
python3 bnovo.py --json
```

### Типы выборки (--type)

| Значение | Что показывает |
|---|---|
| *(без параметра)* | Все брони по дате **создания** в период |
| `checkmate` | Брони, у которых **проживание пересекается** с периодом (как шахматка) |
| `new` | Новые брони, созданные в период |
| `living` | Проживающие в период |
| `checkedIn` | Заехавшие в период |
| `checkedOut` | Выехавшие в период |
| `cancelled` | Отменённые в период |
| `changed` | Изменённые в период |

**Важно:** для расчёта загрузки, ADR и RevPAR всегда использовать `--type checkmate` — только тогда метрики считаются по фактическому проживанию.

## Как работает токен

1. При первом запуске с `--id` и `--password` запрашивает JWT токен через `POST /api/v1/auth`
2. Сохраняет `access_token` и `user_id` в `.bnovo_token.json` с правами 600
3. При следующих запусках декодирует `exp` из JWT и проверяет — если до истечения > 5 минут, использует кэш
4. Если токен истёк — автоматически получает новый, только если API-ключ передан через `BNOVO_PASSWORD` или `--password`

API-ключ / пароль намеренно не пишется в `.bnovo_token.json`, чтобы кэш токена можно было держать рядом со скриптом без риска сохранить долгоживущий секрет в git-репозитории.

## Как считать метрики

Все метрики считаются из массива `prices[]` каждого бронирования:
- Каждый элемент `prices[]` = одна занятая ночь в одной комнате
- `prices[].date` — конкретная дата ночи
- `prices[].price` — цена за эту ночь
- `prices[].room_type_name` — тип дома
- `prices[].room_id` — ID конкретной комнаты

```python
# Формулы
available_nights = rooms_count * period_days
occupied_nights  = len([p for b in bookings for p in b["prices"] if p["date"] in period])
revenue          = sum(p["price"] for b in bookings for p in b["prices"] if p["date"] in period)

occupancy = occupied_nights / available_nights * 100  # %
adr       = revenue / occupied_nights                  # средняя цена ночи
revpar    = revenue / available_nights                 # доход на доступную ночь
```

## Структура ответа API

Каждое бронирование содержит:
- `id`, `number` — идентификаторы
- `status.name` — статус (Новое, Заселен, Выехал, Отменено)
- `source.name` — источник (Прямое, Модуль бронирования, Яндекс Путешествия, Авито, Суточно.ру)
- `customer` — имя, фамилия, телефон, email заказчика
- `dates.arrival` / `dates.departure` — даты заезда/выезда
- `dates.create_date` — дата создания брони (для расчёта booking window)
- `amount` — общая сумма брони
- `prices[]` — разбивка по ночам и комнатам
- `room_type_id`, `room_id` — идентификаторы типа и конкретной комнаты

## Лимиты API

- Авторизация: 10 запросов/минуту
- Остальные: 30 за 10 сек / 300 за 5 мин / 1000 за час / 10 000 в сутки
- При превышении — 429 Too Many Requests

## Тариф API

Текущий тариф — **«Старт»** (бесплатный). Доступны только:
- `POST /api/v1/auth` — авторизация
- `GET /api/v1/bookings` — список бронирований
- `GET /api/v1/bookings/{id}` — одно бронирование

Тариф **«Профессионал»** даёт доступ к тарифам, ценам, комнатам, гостям, платежам, вебхукам.
