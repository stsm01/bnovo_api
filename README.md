# Bnovo PMS API — «Золотой Берег»

Скрипт и документация для работы с API системы Bnovo PMS.

**Base URL:** `https://api.pms.bnovo.ru`
**Спецификация:** `https://api.pms.bnovo.ru/docs/openapi.yaml`

---

## Быстрый старт

```bash
# Первый запуск: ключ лучше передавать через переменную окружения
export BNOVO_ID=107993
export BNOVO_PASSWORD="ключ"
python3 bnovo.py

# Следующие запуски — токен из кэша
python3 bnovo.py

# Указать период и тип выборки
python3 bnovo.py --from 2026-03-30 --to 2026-04-05 --type checkmate
```

## Аутентификация

JWT Bearer токен. Получается через `POST /api/v1/auth`; срок жизни берётся из поля `exp` самого JWT.
Скрипт кэширует токен в `.bnovo_token.json` и обновляет автоматически, если передан API-ключ.

**Важно по безопасности:** `.bnovo_token.json` хранит только `access_token` и `user_id`. API-ключ / пароль в кэш не сохраняется. Для обновления истёкшего токена передайте ключ через `BNOVO_PASSWORD` или `--password`.

```bash
# Получить токен вручную
curl -X POST https://api.pms.bnovo.ru/api/v1/auth \
  -H "Content-Type: application/json" \
  -d '{"id": 107993, "password": "ключ"}'
```

Все дальнейшие запросы:
```
Authorization: Bearer <token>
```

## Методы тарифа «Старт» (бесплатный)

### GET /api/v1/bookings — список бронирований

| Параметр | Обязательный | Описание |
|---|---|---|
| `date_from` | да | Дата начала. Формат: `YYYY-MM-DD` |
| `date_to` | да | Дата конца. Формат: `YYYY-MM-DD` |
| `limit` | да | Кол-во записей (1–50) |
| `offset` | да | Сдвиг для пагинации (от 0) |
| `data_type` | нет | Тип выборки (см. ниже) |

**Значения data_type:**

| Значение | Выборка по |
|---|---|
| *(не указан)* | Дата создания брони |
| `checkmate` | Пересечение периода проживания с запросом |
| `new` | Новые брони, созданные в период |
| `living` | Проживающие в период |
| `checkedIn` | Заехавшие в период |
| `checkedOut` | Выехавшие в период |
| `cancelled` | Отменённые в период |
| `changed` | Изменённые в период |

**Для расчёта загрузки и RevPAR всегда использовать `checkmate`.**

### GET /api/v1/bookings/{id} — одно бронирование

| Параметр | Тип | Описание |
|---|---|---|
| `with_linked` | bool | Включить связанные брони |

## Структура бронирования

```json
{
  "id": 92684554,
  "number": "S7E3R-290326",
  "amount": 11656,
  "status": { "id": 1, "name": "Новое" },
  "source": { "name": "Яндекс Путешествия (новая версия)" },
  "customer": {
    "name": "Валерия", "surname": "Холодилова",
    "phone": "79930710322", "email": "..."
  },
  "dates": {
    "create_date": "2026-03-29 10:03:47",
    "arrival": "2026-05-27 16:00:00+03",
    "departure": "2026-05-29 13:00:00+03"
  },
  "prices": [
    {
      "date": "2026-05-27",
      "price": 5828.0,
      "room_type_name": "Бунгало",
      "room_id": 412161,
      "room_type_id": 68
    }
  ],
  "room_id": 68,
  "room_type_id": 70
}
```

**Ключевое:** метрики (загрузка, ADR, RevPAR) считаются из `prices[]` — каждый элемент это одна занятая ночь в одной комнате.

---

## Расчёт метрик

```python
from datetime import date, timedelta

CAPACITY = {
    "Финский Коттедж": 7,
    "Бунгало": 5,
    "A-Frame": 1,
    "Family A-Frame": 3,   # с 2026 года; в 2025 было 2
    "Family aframe2": 1,
}

period_dates = {"2026-03-30", "2026-03-31", ..., "2026-04-05"}  # 7 дат

# Из bookings (checkmate):
occupied = sum(1 for b in bookings
               for p in b["prices"] if p["date"] in period_dates)
revenue  = sum(p["price"] for b in bookings
               for p in b["prices"] if p["date"] in period_dates)
available = sum(CAPACITY.values()) * 7

occupancy = occupied / available * 100  # %
adr       = revenue / occupied           # ₽ средняя цена ночи
revpar    = revenue / available          # ₽ доход на доступную ночь
```

## Лимиты запросов

| Ограничение | Лимит |
|---|---|
| Авторизация `/api/v1/auth` | 10 запросов/минуту |
| Остальные эндпоинты | 30 / 10 сек |
| | 300 / 5 мин |
| | 1 000 / час |
| | 10 000 / сутки |

При превышении → **429 Too Many Requests**

---

## Коды ошибок

| Код | Причина |
|---|---|
| 401 | Токен истёк — повторить авторизацию |
| 404 | Пользователь не найден — проверить ID и пароль |
| 406 | Ошибка валидации параметров |
| 429 | Превышен лимит запросов |

## Полный список методов API

Тариф «Старт» — только GET бронирований (см. выше).
Тариф «Профессионал» добавляет:

| Группа | Методы |
|---|---|
| Бронирования | заказчик, счета, услуги, гости, статус |
| Гости | теги, справочники документов, виз, целей въезда |
| Комнаты | список, закрытые продажи |
| Наличие | доступные комнаты и типы |
| Тарифы | список тарифов, цены по тарифу |
| Платежи | список, создание, детали |
| Отчёты | ADR/RevPAR |
| Вебхуки | подписчики, тестовые события |
| Справочники | страны, юрлица, платёжные системы |
